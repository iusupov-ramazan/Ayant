# Ayta — что нового в этой версии

Локальная платформа открытий для Центральной Азии (старт — Бишкек).
Две стороны в одном приложении: **пользователь** и **заведение (хост)**. Переключение
между ними — без отдельного приложения и повторного входа.

---

## Брендинг

- Приложение переименовано в **Ayta** (название под иконкой, заголовки, экраны входа,
  коды предложений `AYTA-…`, тексты «Нашёл в Ayta!», админ-панель «Ayta Админ»).

---

## Пользовательская сторона

### Онбординг (при первом запуске)
- Пошаговый онбординг: **город** (пока только Бишкек), запрос **геолокации**
  («при использовании») и запрос **уведомлений**. Без геолокации приложение работает,
  только без расстояний.

### Главная — лента предложений
- Лента теперь показывает **предложения (акции)**, а не заведения.
- Карточка предложения: фото, заведение (с галочкой «проверено»), скидка, заголовок,
  описание, цена, кнопки **сохранить** и **поделиться** (диплинк).
- Верхний ряд **категорий**-кружков (как сторис) фильтрует ленту.
- Лента **«Сегодня в избранном»** — специалы сохранённых заведений.
- Каждый 5-й слот — **рекламный** («Здесь может быть ваша реклама»).
- Тап по предложению **открывает экран деталей пушем** (слайд с кнопкой «назад»),
  а не модально снизу.
- Город: вся лента и поиск ограничены выбранным городом; показываются только
  **одобренные** и **не на паузе** заведения.

### Поиск
- Полнотекстовый поиск по названию, категории, акции и району.
- Фильтры: **открыто сейчас**, **минимальный рейтинг** (★3+/★4+),
  **расстояние** (500 м / 1 / 3 / 5 км), **категория**.

### Сохранённое
- Две вкладки: **Заведения** и **Предложения**. Свайп для удаления.

### Профиль
- Аватар, имя, город. **Мои отзывы** (редактировать/удалить). Настройки:
  язык (рус/узб/англ), уведомления, тема. Кнопка **«Переключиться в режим заведения»**.
  Выход и удаление аккаунта.

### Страница заведения
- Большое **фото-обложка** (если есть) + аватар-логотип; **без эмодзи** — при отсутствии
  фото показывается чистый фон со значком.
- Название + **галочка «Проверено»**, категория, **агрегированный рейтинг + число отзывов**,
  бейдж «Сегодня», строка «Сохранили N человек».
- Действия: **Позвонить · Маршрут · Сохранить · Поделиться** (диплинк).
- Адрес с кнопками **2GIS** и **Google Maps**, телефон, **часы работы по дням недели**
  (сегодня выделено), **прайс-лист / каталог (PDF)** во встроенном просмотрщике, карта-пин.
- **Сетка предложений** (3 колонки), **галерея реальных фото** (обложка + фото объектов +
  фото из отзывов) с полноэкранным просмотром и жалобой.
- **Объекты для отзывов** (блюда/услуги) с фото — тап открывает форму отзыва на объект.

### Отзывы
- **Отзыв всегда на конкретный объект** (блюдо/услугу), а не на заведение в целом.
- Звёзды, текст и **до 3 реальных фото** (загрузка с телефона).
- Разбивка рейтинга 5★…1★, ответы владельца («Ответ заведения»), жалоба в 2 тапа.
- Отзывы и ответы публикуются в Firestore — видны всем и на других устройствах.

### Прочее
- **Диплинки**: `san://venue/<id>`, `san://deal/<id>` и Universal Links
  `https://<домен>/venue|deal/<id>`. Шаринг отправляет ссылку, тап по рекламному push
  открывает нужное заведение.
- **Гость (анонимный вход)** — только просмотр: не может создавать заведения, сохранять
  и оставлять отзывы (подсказки войти).
- Вкладка **«Бонусы»** с игрой «Змейка» (активное время → бонусы).

## Сторона заведения (хост)

5 вкладок: **Заведения · Продвижение · Аналитика · Отзывы · Профиль**.

### Онбординг хоста
- Запускается по «Переключиться в режим заведения». Шаг 1 — данные бизнеса
  (название, категория, телефон, email). Шаг 2 — добавить первое заведение или позже.

### Мои заведения
- Список заведений с **фото**, статусом модерации и активностью.
- Создание/редактирование заведения:
  - Название, категория, район, адрес, телефон.
  - **Фото заведения** (загрузка с телефона).
  - **Прайс-лист / каталог (PDF)** — загрузка PDF со списком блюд/услуг.
  - **Локация на карте** (выбор точки) или ручной ввод координат.
  - **Часы работы по каждому дню недели** (время открытия/закрытия, выходной;
    кнопка «применить понедельник ко всем дням»).
  - **Объекты для отзывов** (блюда/услуги) — название, тип, **фото**.
  - **Сегодняшний специал** (одной строкой).
  - Переключатель «активно / на паузе».
- **Предложения**: сетка с действиями **Изменить · Пауза · Дублировать · Удалить**,
  загрузка **фото** к предложению, статусы (активно/пауза/завершено/черновик).
- Контент хоста хранится в Firestore (с владельцем `ownerID`) и появляется в
  пользовательской ленте; локальный кэш — офлайн-фолбэк.

### Продвижение
- Кампании по всем заведениям. Два типа:
  - **Буст заведения** (7/14/30 дней, цена заранее).
  - **Push-уведомление** (заголовок ≤60, текст ≤120) — реальная рассылка.

### Аналитика
- Период 7/30/90 дней. **Реальные данные из Firestore**: просмотры профиля, клики по
  предложениям, сохранения, звонки, маршруты — агрегировано и по каждому заведению.

### Отзывы
- Инбокс отзывов по всем заведениям, бейдж непрочитанных. Публичный ответ владельца.

### Профиль заведения
- Данные бизнеса, **запрос галочки «Проверено»** (статус), уведомления, оплата,
  «Вернуться в режим пользователя».

## Модерация и верификация

- **Новое заведение создаётся со статусом «на модерации»** и не видно пользователям,
  пока не одобрено. Хост видит статус (потяните, чтобы обновить).
- **Админ-панель** (веб, GitHub Pages): одобрение/отклонение заведений, **верификация
  заведений/бизнесов** (галочка), управление объектами, фото, статусом.

## Бэкенд и инфраструктура

- **Firestore** коллекции: `venues`, `deals`, `reviews`, `hosts`, `analytics`,
  `pushCampaigns`, `userTokens`, `pushLog`. Правила доступа опубликованы.
- **Cloud Function** `sendPushCampaign`: адресная рассылка push по токенам города с
  **частотным лимитом** — не более 1/день и 3/неделю на пользователя.
- **Хранение изображений — Cloudinary** (unsigned-загрузка с телефона и из админки;
  авто-сжатие `c_limit,w_1600,q_auto`). PDF-каталоги — туда же.
- **Админ-панель** на GitHub Pages (папка `docs/`).
- Регистрация FCM-токенов устройств, обработка тапа по push (диплинк), Universal Links
  (entitlement + AASA-шаблон).

---

## Технические заметки

- Один аккаунт = один профиль хоста = одно или несколько заведений.
- Часы работы хранятся по дням недели (минуты от полуночи), с фолбэком на единое время.
- Изображения и PDF хранятся как ссылки (URL) в Firestore; файлы — в Cloudinary.
- Для доставки PDF включите в Cloudinary: **Settings → Security → Allow delivery of PDF and ZIP files**.
