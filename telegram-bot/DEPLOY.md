# Деплой бота 24/7 (бесплатно, без терминала)

Бот будет крутиться в облаке Render и работать круглосуточно — твой компьютер не нужен.
Ключи (токен бота, Groq) хранятся в дашборде Render, **не в коде**.

---

## Шаг 1. Залить код на GitHub

Папка `telegram-bot/` уже в твоём репозитории `Yelp`. Убедись, что репозиторий запушен на GitHub.
⚠️ Файл `.env` **не** должен попасть в git (он уже в `.gitignore`). Ключи зададим в Render вручную.

## Шаг 2. Создать сервис на Render

1. Зайди на **render.com** → войди через GitHub (бесплатно, без карты).
2. **New → Blueprint** → выбери репозиторий `Yelp`.
   Render найдёт `telegram-bot/render.yaml` и предложит создать сервис `ayta-support-bot`.
   (Если Blueprint не видит файл — выбери **New → Web Service**, репозиторий `Yelp`,
   Root Directory: `telegram-bot`, Build: `npm install`, Start: `npm start`, Plan: **Free**.)
3. На шаге переменных окружения впиши:
   - `TELEGRAM_BOT_TOKEN` → токен от @BotFather
   - `GROQ_API_KEY` → ключ с console.groq.com
   (`GROQ_MODEL` и `SUPPORT_CONTACT` уже заданы по умолчанию.)
4. **Create** → подожди 1–2 минуты. В логах должно появиться:
   `🌐 health-check на порту ...` и `✅ @ТвойБот запущен`.

Проверь: открой бот в Telegram, напиши вопрос — должен ответить.

## Шаг 3. Чтобы не «засыпал» (важно для бесплатного плана)

Бесплатный сервис Render засыпает после 15 минут без обращений. Держим его живым бесплатным пингером:

1. Скопируй URL сервиса из Render (вид `https://ayta-support-bot.onrender.com`).
2. Зайди на **uptimerobot.com** (бесплатно) → **Add New Monitor**:
   - Type: **HTTP(s)**
   - URL: вставь адрес сервиса
   - Interval: **5 минут**
3. Save. Теперь UptimeRobot стучится каждые 5 минут, и бот не засыпает.

   (Альтернатива: **cron-job.org** — тоже бесплатно, то же самое.)

## Готово

Бот работает 24/7. Когда меняешь тексты (`faq.js`) или код — просто пушишь в GitHub,
Render пересобирает автоматически.

### Если что-то не так
- В Render → твой сервис → **Logs**. Ошибка `401` = неверный токен/ключ; `404` = неверная модель (поменяй `GROQ_MODEL`).
- Бот молчит после паузы → проверь, что монитор UptimeRobot активен (зелёный).
