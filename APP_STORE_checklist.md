# Ayant — App Store / TestFlight чек-лист

Полный список того, что нужно для загрузки в TestFlight и публикации в App Store.
Технические требования актуальны на 2026 г.

---

## 1. Иконка приложения

- **1024 × 1024 px**, PNG, **без прозрачности** (no alpha), без скруглённых углов (Apple скруглит сама), RGB.
- Простая, узнаваемая: логотип «Ayant» / буква на фирменном оранжевом (#FF4D29).
- Добавляется в Xcode → `Assets.xcassets` → AppIcon (или в App Store Connect).

## 2. Скриншоты (обязательно)

Минимум **3–5 скриншотов** для основного размера. Точные пиксели — иначе отклонят (даже ±1 px).

| Устройство | Размер (px) | Статус |
|---|---|---|
| 6.9″ (iPhone 17 Pro Max) | **1320 × 2868** | ✅ ведущий — делай этот |
| 6.7″ (iPhone Plus/15 Pro Max) | 1290 × 2796 | принимается как fallback |
| 6.5″ | 1284 × 2778 | опционально |
| iPad 13″ | 2064 × 2752 | только если поддерживаешь iPad |

PNG/JPEG, RGB, без alpha. Сделать проще всего: запусти на симуляторе iPhone 16/17 Pro Max → ⌘S (скриншот точного размера).

**Что показать на 5 скриншотах:**
1. Лента САН (скидки/акции/новинки) — «Все скидки города в одной ленте».
2. Страница заведения (фото, часы, акции) — «Вся информация о заведении».
3. Карта заведений — «Находи места рядом».
4. Бонусы и игры — «Играй и копи бонусы».
5. Купон с QR — «Меняй бонусы на скидки».

## 4. Конфиденциальность (App Privacy в App Store Connect)

Заполни анкету «Data Collection» так (что реально собираешь):
- **Contact Info** → Email, Name — связано с пользователем, для работы приложения.
- **Location** → Coarse Location — для работы приложения (расстояние).
- **Identifiers** → Device ID (push-токен) — для уведомлений.
- **Usage Data** → Product Interaction — аналитика (Firebase).
- **User Content** → Photos, Other (отзывы) — для работы приложения.
- Данные **не** используются для трекинга через рекламные сети (App Tracking Transparency не нужен).

## 5. TestFlight — шаги загрузки

1. В Xcode выбери устройство **Any iOS Device (arm64)**.
2. **Product → Archive**.
3. В окне Organizer → **Distribute App** → **App Store Connect** → **Upload**.
4. Export compliance: в `Info.plist` уже стоит `ITSAppUsesNonExemptEncryption = false` → выберешь «нет шифрования».
5. После обработки билда (10–30 мин): App Store Connect → **TestFlight**.
6. **Внутренние тестеры** (до 100, мгновенно) — добавь себя и команду.
7. **Внешние тестеры** (до 10 000) — создай группу, добавь email; первый внешний билд проходит короткий Beta App Review.
8. Тестеры ставят приложение **TestFlight** и заходят по приглашению.

## 6. Для App Review (когда подаёшь на публикацию)

- **Демо-доступ:** в приложении есть режим **гостя** — укажи это в «App Review Information → Notes», чтобы рецензент мог войти без аккаунта. Либо заведи тестовый email-аккаунт и дай логин/пароль.
- **Notes для рецензента (пример):**
  ```
  Войдите как гость (кнопка «Продолжить как гость») или используйте тестовый аккаунт.
  Приложение — каталог скидок заведений Бишкека. Геолокация используется только для
  показа расстояния. Купоны/бонусы — внутренняя механика, без реальных платежей.
  ```
- Убедись, что в билде для релиза **убран** `?mode=developer` из Associated Domains (он только для разработки).
